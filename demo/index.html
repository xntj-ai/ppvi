PPVI

— a system of restraint

设计的本质
不是加法,
是选择。

PPVI 是一套经过实战验证的设计美学方法论。它不教你"加什么让页面好看", 而是回答"为什么减到这里就够了"。每一个像素都是一次审美决策, 每一条原则背后都是一次取舍。

00 / Modes

同一套逻辑,两套表达

Same system, two surfaces.

浅色系 (#fafafa) 和深色系 (#0a0a0a) 不是两套独立的设计,而是同一套 token 在两种环境下的映射。背景层数、字重梯度、留白节奏、玻璃透明度——结构完全一致, 只是色值反转。不存在"专属于深色"或"专属于浅色"的好设计, 只存在"用同一套逻辑做出来的两种皮肤"。

克制优雅

Less, but deliberate.

好设计不是"加到好看",是"减到刚好"。每个视觉元素必须有存在理由, 问不出理由就删掉。装饰 ≠ 美。一个精心选择的渐变 > 十个随意的阴影。

Decision rule

问不出存在理由,就删掉。

Anti
  • 渐变彩虹色 — 没有审美判断的堆砌
  • 随机阴影叠加
  • 无理由的圆角变化
  • 为动画而动画
反例 · 加法思维
5 个动作 / 5 种颜色 / 0 个层级
正例 · 减法思维
5 个动作 / 2 种权重 / 1 处强调

灰阶为基

Color is punctuation, not prose.

灰阶是画布,颜色是标点。大面积用灰阶建立层次,色彩只用在需要注意力的地方。 彩色占比 ≤ 15%,一个主色 + 一个强调色足矣——不需要"调色盘"。

Ratio

85% 灰阶 · 15% 强调色

灰阶承担结构 彩色承担注意力

03 / 05

严格层级

Every size has a reason.

字号不是"大中小",是有严格定义的 5 级系统。跨级使用,级内一致。 单视图字号数量上限 5 个,跨级比例 ≥ 1.4x。强调用字重,不用字号变化。

5 vectors of emphasis
  • Scale — 字号
  • Weight — 字重
  • Spacing — 行距/字距
  • Tracking — 字间距
  • Alignment — 对齐

至少 2 个 vector 同向才算 dominant。 单一 vector 的对比不够"硬",层级会发软。

04 / 05

玻璃拟态

Depth through light.

玻璃效果不是"加个模糊",是用光、透明度、层叠创造空间纵深感。 三层结构缺一不可:背景层 (暗/纹理) → 玻璃层 (半透明 + 模糊) → 内容层 (清晰文字)。

Three layers, in order
L1 背景层暗色 + 噪点 + 微妙渐变 noise 0.05
L2 玻璃层半透明 + backdrop-blur blur 48px
L3 内容层清晰文字 + 高对比 opacity 1

暗底 + 亮玻璃 > 亮底 + 暗玻璃。 深色背景上的浅色玻璃,层次比反过来更分明。

L2 · GLASS 玻璃在彩色背景上才显形 把这块玻璃放到纯灰背景上,它就消失了。 玻璃需要被"映"出来——背景有色彩信息,玻璃才有可被模糊的内容, 才能体现"前面有一层东西"的纵深。
05 / 05

密度呼吸

Every pixel of whitespace is intentional.

不是"留白越多越好",是信息密度和呼吸感的精确平衡。 相关元素紧密,无关元素远离——卡片内紧凑 (12-16px),卡片间宽松 (24-32px)。

Gestalt · 亲近性

视线把距离近的元素认作一组,把距离远的元素认作不同组。 所以留白本身就是分组——比边框、分隔线、背景色更优雅。

Rule of thumb
  • 卡片内 padding: 12–16px
  • 卡片间 gap: 24–32px
  • 区块间隔: 80–160px
  • 滚动条/分隔线: 越少越好

审美决策框架

Five questions, in order.

面对任何一个设计选择时,按这五个问题顺序判断。 前面的问题没过,后面的就不用问——决策有优先级,跳级是错的。

01
有存在理由吗?Does it earn its pixels?
→ 没有就删掉
02
层级清晰吗?Can the eye find the priority in 0.5s?
→ 跨级 ≥ 1.4x
03
色彩克制吗?Does color earn the attention?
→ 彩色 ≤ 15%
04
有纵深感吗?Is the interface layered, not flat?
→ 三层结构
05
呼吸感对吗?Are tight things tight, loose things loose?
→ 12 内 · 24 间
// Anti-patterns

绝对避免

If you see these, the system has already failed.

反模式不是品味问题,是系统失效的证据—— 它意味着某一处没有做决策,而是用了默认值。

×渐变彩虹背景
紫→粉→蓝的"AI 感"渐变,没有任何审美判断,是默认 SaaS landing page 的视觉投降。
失败点 · 把"看起来现代"当成审美
×所有元素等大等重
没有层级就没有设计。读者不知道先看哪里,所有信息一起喊,等于没人说话。
失败点 · 怕做减法
×用边框分隔一切
间距和背景色差异是更优雅的分隔。边框是补丁,告诉用户"我不会做留白"。
失败点 · 把视觉噪音当结构
×纯白大面积背景
#ffffff 太刺眼。用 #fafafa 或带微妙噪点的灰白,眼睛舒服 10 倍。
失败点 · 把"默认"当"正确"
×Inter + 紫色 + 白底
通用 AI 审美。零辨识度。一千个 SaaS 长一样。PPVI 用系统字体 + 灰阶 + 单一赭红故意拒绝它。
失败点 · 没有立场
×为动画而动画
动画是为了传达层级变化和状态。slide-in / fade-up 的炫技堆叠,是设计师在替自己表演。
失败点 · 服务自己而非用户
Restraint
is a stance.

克制不是风格选择,是立场。它说:"我知道可以加,我选择不加。" 这才是 PPVI 想守护的东西。
