ppvi is a Claude Code / Agent skill — not code, a way of thinking. Install it and every page Claude builds is produced under one taste framework: five core principles, a dark/light dual system, and full-screen / half-screen / mobile adaptation.

Claude Code's design ability is already strong, but left alone the output carries the "generic AI" tell — Inter font, purple gradient, pure-white background, random rounded corners. The problem isn't skill, it's the absence of taste constraints: anything is possible, so nothing is decided. ppvi supplies the missing layer, telling the model why this color, this spacing, this weight — so taste becomes reproducible instead of accidental.

The five principles

Five rules distilled over six months of real sharing pages — the structural backbone of every ppvi design.

1 · Restraint 克制优雅

Design is subtraction. Every element must justify itself — if you can't name a reason, delete it. One deliberate gradient beats ten random shadows.

2 · Grayscale base 灰阶为基

Grayscale is the canvas, color is the punctuation. Large areas stay neutral; color (≤ 15%) appears only where attention belongs.

3 · Strict hierarchy 严格层级

Not "big / medium / small" — a defined 5-level scale, ≥ 1.4× between levels, consistent within. Emphasis comes from weight, not size.

4 · Glassmorphism 玻璃拟态

Not "add a blur" — light, transparency and stacking that create real depth. Background → glass layer (blur 32–64px) → crisp content.

5 · Density & breath 密度呼吸

Whitespace is functional, not empty. Tight inside, loose between — card padding 12–16px, card gap 24–32px. Islands, with ocean around them.

+ Two execution rules 执行规则

Responsive — fluid clamp() type, container queries, Chinese line-breaking. Diagram, don't paragraph — an 11-type chart catalog with hypothesis-driven titles.

Strict hierarchy, visualized

A 5-level type scale and a grayscale ramp — the two systems behind "looks ordered" before any color is added.

5 sizes max per view ≥ 1.4× between levels emphasis by weight, not size

Hierarchy isn't size alone. ppvi builds it from five vectors — Scale · Weight · Spacing · Tracking · Alignment — and a dominant element needs at least two of them pulling the same way. Weight runs on three steps: 400 read / 510–550 emphasize / 590–700 announce.

ppvi vs frontend-design

Two skills, two layers — complementary, not competing. Install both and let taste and technique work as one pair of hands.

ppvi · How to think

Aesthetic decisions

  • Why this color, this spacing, this weight
  • Whether an element earns its place
  • Whether the hierarchy reads in 0.5s
  • Output: a taste framework + checklist
frontend-design · How to write

Technical execution

  • How to code, animate, Tailwind it
  • Component structure and state
  • Production-grade, ships to the browser
  • Output: real, working code

ppvi decides what is right; frontend-design makes it real. Run the design through ppvi's framework first, then hand the decisions to frontend-design to build.

Core ideas

Methodology, not code

No library to import, no build step — a set of taste rules Claude reads and applies to whatever it builds (HTML/CSS, a React component, a report page).

Dark / light dual system

A full dark and light scope. Dark shows depth most clearly; most xntj.tv sharing pages run light-mode ppvi.

One accent, used well

A single accent color (the demo's is #d97757) carries all the attention; grayscale does everything else. Color is punctuation, not prose.

Anti-pattern aware

An explicit list of tells to avoid — Inter-only fonts, purple gradients, pure-white fields, random corner values, Excel-default charts.

When to use

Good fit

When taste is the decision

  • Sharing pages, reports, dashboards, landing pages, posters
  • Building a visual system or defining a UI style
  • Any "looks good but I can't say why" call
Not for

When it's not a taste problem

  • Pure technical implementation — use frontend-design
  • Brand logo design — needs a brand-identity process
  • Print / CMYK — ppvi targets screens

Quick start

Install into your Claude Code skills directory

git clone https://github.com/xntj-ai/ppvi.git ~/.claude/skills/ppvi

Ask Claude to apply ppvi

For example, "use ppvi to design this report page," or "apply ppvi and tell me why this layout works" — it runs the five-principle decision framework on your design.

Pair it with frontend-design

Let ppvi decide what's right (taste), then hand the decisions to frontend-design to build (technique). Open demo/index.html to see every principle applied at once.
