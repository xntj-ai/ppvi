ppvi 是一个 Claude Code / Agent 技能 —— 它不是代码,是一套"怎么想"的方法。装上之后,Claude 做出来的每一个页面都按同一套审美框架生产:五条核心原则、深浅双系统、全屏/半屏/手机自适应。

Claude Code 的设计能力已经很强,但放任不管,出来的页面有"通用 AI"的味道 —— Inter 字体、紫色渐变、纯白底、随机圆角。问题不在能力不够,而在缺少审美的约束:什么都能做,于是什么都没决定。ppvi 补上缺的那一层,告诉模型为什么选这个颜色、这个间距、这个字重 —— 让审美变得可复现,而不是碰运气。

五条核心原则

在半年实战分享页里沉淀出的五条规则 —— 每一个 ppvi 设计的结构骨架。

1 · 克制优雅 Restraint

设计是减法。每个元素必须有存在理由 —— 问不出理由就删掉。一个精心选择的渐变,胜过十个随意的阴影。

2 · 灰阶为基 Grayscale

灰阶是画布,颜色是标点。大面积保持中性,色彩(≤ 15%)只出现在需要注意力的地方。

3 · 严格层级 Hierarchy

不是"大中小" —— 是有定义的 5 级字号系统,跨级 ≥ 1.4×,级内一致。强调用字重,不用字号。

4 · 玻璃拟态 Glass

不是"加个模糊" —— 是用光、透明度、层叠创造真实纵深。背景层 → 玻璃层(模糊 32–64px)→ 清晰内容。

5 · 密度呼吸 Density

留白是有功能的,不是空。内紧外松 —— 卡片内 padding 12–16px,卡片间 gap 24–32px。每个功能区是一座岛,岛与岛之间有海洋。

+ 两条执行规则 Execution

响应自适 —— fluid clamp() 字号、容器查询、中文换行。图示穿插 —— 11 类图表 catalog,标题假设驱动,不堆段落。

严格层级,可视化

一套 5 级字号系统,加一条灰阶梯度 —— 还没上色,"看起来有序"就已经成立。

单视图最多 5 个字号 跨级 ≥ 1.4× 强调用字重不用字号

层级不只靠字号。ppvi 用五个维度建立它 —— 字号 · 字重 · 间距 · 字距 · 对齐 —— 一个主导元素至少要两个维度同向用力才算"突出"。字重走三档:400 阅读 / 510–550 强调 / 590–700 宣告。

ppvi 与 frontend-design

两个技能,两个层 —— 互补,不是二选一。同时装上,让审美与技术当成一双手来用。

ppvi · 怎么想

审美决策

  • 为什么选这个颜色、这个间距、这个字重
  • 一个元素配不配占这个位置
  • 层级能不能 0.5 秒内读出来
  • 产物:一套审美框架 + 清单
frontend-design · 怎么写

技术执行

  • 怎么写代码、做动画、用 Tailwind
  • 组件结构与状态
  • 可上线品质,直接跑进浏览器
  • 产物:真正能跑的代码

ppvi 决定什么是对的,frontend-design 把它变成真的。先用 ppvi 的框架过一遍设计,再把决策交给 frontend-design 落地。

核心理念

方法论,不是代码

没有库要导入、没有构建步骤 —— 一组 Claude 读了就照着做的审美规则,体现在它正在做的东西里(HTML/CSS、React 组件、报告页)。

深浅双系统

完整的暗色与浅色两套作用域。暗色纵深感最分明;xntj.tv 的大部分分享页跑的是浅色 ppvi。

一个强调色,用到位

单一强调色(demo 用的是 #d97757)承载全部注意力,灰阶做其余一切。颜色是标点,不是正文。

反模式自觉

一份明确的 tell 清单要避开 —— Inter 当唯一字体、紫色渐变、纯白大面积底色、随机圆角值、Excel 默认样式图表。

适用场景

适合

当审美就是那个决策

  • 分享页、报告、仪表盘、落地页、海报
  • 建立视觉系统,或定义 UI 风格
  • 任何"好看但说不清为什么"的取舍
不适合

当它不是审美问题

  • 纯技术实现 —— 用 frontend-design
  • 品牌 logo 设计 —— 需专门的 brand identity 流程
  • 印刷 / CMYK —— ppvi 面向屏幕

快速开始

装进 Claude Code 技能目录

git clone https://github.com/xntj-ai/ppvi.git ~/.claude/skills/ppvi

让 Claude 用 ppvi

比如"用 ppvi 帮我设计这个报告页",或"用 ppvi 审一下这个布局,告诉我为什么成立" —— 它会把那套五原则决策框架跑在你的设计上。

和 frontend-design 搭配

让 ppvi 决定什么是对的(审美),再把决策交给 frontend-design 落地(技术)。打开 demo/index.html 一次看全所有原则的应用。
