# 图表与图示 Catalog

## 为什么需要图表 Catalog

文字传达逻辑慢,图表传达关系快。但 AI 默认会把所有信息排成段落 + 列表,失去视觉穿插的节奏。

本 catalog 给出 11 类常用图表类型 + 何时选用,让"信息→图表"的判断有据可依。

## 标题前置规则(硬约束)

每个图表的标题 MUST 是 **假设驱动**,不是描述性。

| ❌ 描述性 | ✅ 假设驱动 |
|---|---|
| "AI 视频模型对比" | "时序一致性成为关键差异" |
| "2024 年营收数据" | "Q3 单季增速首次反超头部对手" |
| "团队结构" | "三层管理压缩为两层后决策提速" |

判断标准:把标题单独拿出来,读者是否能立即知道**结论**? 不能 → 改写。

## 11 类图表类型

### 1. Time-Series Growth(时序增长)

- **Use when**: 单一指标随时间变化,强调趋势
- **Elements**: x 轴时间(月/季/年)/ y 轴指标 / 起止点强调 / 关键拐点标注
- **PPVI 适配**: 单色折线 + 灰阶网格, 拐点用强调色圆点, 区域填充 opacity 0.08

### 2. Gap / Funnel(漏斗/落差)

- **Use when**: 转化流失,或目标 vs 实际差距
- **Elements**: 从大到小逐层缩窄 / 每层数值 + 转化率 / 最关键流失层用强调色
- **PPVI 适配**: 阶梯式色块灰阶递减, 流失百分比用 Caption 级灰字附在右侧

### 3. Before / After(改造对比)

- **Use when**: 单一改动带来的前后变化
- **Elements**: 左右两栏对称 / 中间分隔不超 2px / "Before" "After" 标签明确 / 数字差异用强调色
- **PPVI 适配**: 左栏背景 #f5f5f5, 右栏背景 #fafafa, 强调色仅用在差值数字上

### 4. Market Share Donut(份额构成)

- **Use when**: 整体由几部分组成, 强调相对占比
- **Elements**: 不超 6 块(超过用堆叠条形图) / 中心放总量数字 / 最大块用强调色, 其余用灰阶
- **PPVI 适配**: 环宽不超直径 1/4, 中心数字用 Headline 级, 标签用 Caption 引线连接

### 5. Investment / Scale Infographic(投入/规模信息图)

- **Use when**: 双栏对比两个对象的多维属性
- **Elements**: 左右两栏 / 每栏 3-5 个属性 / 数字大字 + 单位小字 / 图标可选(图标必须同源)
- **PPVI 适配**: 不用装饰图标, 用 Subtitle 级数字 + Caption 级单位 + 1px 分隔线

### 6. Timeline(时间线)

- **Use when**: 多个事件的时间顺序
- **Elements**: 水平或垂直主轴 / 节点圆点 / 节点旁日期 + 事件名 / 关键节点放大
- **PPVI 适配**: 主轴 1px 灰线, 节点 8px 圆点, 关键节点 12px + 强调色

### 7. Comparison Split(二选一对照)

- **Use when**: 两个方案/概念的逐项对比
- **Elements**: 垂直分割 / 每行一个维度 / 维度名居中 / 左右各填内容
- **PPVI 适配**: 分割线 1px opacity 0.1, 维度名用 Caption 灰字, 内容用 Body, 优胜项加强调色边框 2px

### 8. 2×2 Strategic Framework(战略矩阵)

- **Use when**: 用两个维度划分四个象限
- **Elements**: 横轴+纵轴标签 / 四象限标签 / 项目散点 / 推荐象限用强调色背景
- **PPVI 适配**: 象限分隔线 1px, 推荐象限背景 opacity 0.05 强调色, 散点用 Caption 级文字

### 9. Benchmarking Table(竞品对比表)

- **Use when**: 多对象 × 多维度的全面对比
- **Elements**: 表头加粗 / 行间 hairline border / 关键单元格高亮 / 不超 5 列
- **PPVI 适配**: border 用 `rgba(0,0,0,0.06)`, 表头背景 #f5f5f5, 优胜单元格用强调色文字而非背景

### 10. Waterfall(瀑布图,增减归因)

- **Use when**: 起始值 → 多步增减 → 终值,归因每一步贡献
- **Elements**: 柱状垂直排列 / 增量用一色减量用另一色 / 起始与终值用第三色 / 累计值连线
- **PPVI 适配**: 增量灰深, 减量灰浅, 起止用强调色, 连线 1px opacity 0.2

### 11. Cover Slide(封面)

- **Use when**: 报告/演示首页
- **Elements**: Display 标题 / Subtitle 副标题 / 时间地点元数据 / 大留白
- **PPVI 适配**: 标题占左下或左上 1/3 区域, 其余留白, 元数据用 Caption 灰字置底

## 逻辑图穿插策略

不只是数据图——逻辑/流程也需要图示化。

| 场景 | 用什么 | 别用什么 |
|---|---|---|
| 因果链 | Timeline 横向 + 箭头 | 纯文字 "A 导致 B 导致 C" |
| 决策分支 | 树形图(2-3 层) | 嵌套 if-else 文字 |
| 系统架构 | 分层方框 + 1px 连线 | 全文字 + bullet |
| 流程步骤 | 编号 + 横向流转 | 数字列表 |
| 概念关系 | 2×2 或 Venn | 段落描述 |

## 信息密度上限(单图表)

| 图表类型 | 数据点上限 | 文字标注上限 |
|---|---|---|
| 折线/柱状 | 12 个数据点 | 4 处标注 |
| 饼/环 | 6 块 | 6 个标签 |
| 表格 | 7 行 × 5 列 | 不限(简短) |
| 流程图 | 7 步 | 每步 ≤8 字 |
| 2×2 | 8 个散点 | 4 个象限标签 |

超过上限 → 拆分多张图, 或换更合适的图表类型。

## 颜色规则(图表专用)

- **单系列**: 用 PPVI 主色, 灰阶辅助
- **双系列对比**: 主色 + 灰阶, 不要双彩色
- **多系列(3+)**: 同色相不同明度梯度, 不要多色相
- **强调单点**: 全图灰阶, 仅强调点用主色

## 反模式

- ❌ **3D 图表** — 视觉扭曲数据
- ❌ **装饰性图标** — 用文字, 不用 emoji
- ❌ **彩虹色梯度** — 用灰阶+单一强调色
- ❌ **图例与图分离过远** — 图例就近放, 或直接标在数据旁
- ❌ **描述性标题** — 见本文顶部硬约束
- ❌ **复制粘贴 Excel 默认样式** — 网格线、刻度、图例都要重新设计

## 工具选型建议

- **Web 端静态图**: SVG 直接写, 或 D3 / Observable Plot
- **Web 端交互图**: ECharts / Recharts(主题需自定义, 默认样式不合规)
- **海报内图表**: SVG 嵌入, 不用 PNG
- **逻辑图/流程图**: 优先手写 SVG, 复杂时用 Mermaid + 自定义主题, 避免默认蓝白配色
