# 色彩哲学：灰阶为基

## 核心观点

大多数"配色不好看"的设计，问题不在颜色选择，而在颜色太多。

灰阶是最安全的大面积用色。彩色只在需要用户注意的地方出现 — 按钮、状态标识、关键数据。这样每一处彩色都有意义，用户眼球自然被引导到正确的位置。

## 灰阶画布

### 背景灰度梯度

不要用纯白 (#ffffff) 做大面积背景。微妙的灰度差异创造层次，不需要边框和阴影。

```
最底层背景:    #fafafa  (98% 亮度) — 页面底色
卡片/面板:     #ffffff  (100%) — 在灰底上自然凸起
侧边栏:        #f5f5f5  (96%) — 比背景略深，划分区域
工具栏:        #e0e0e0  (88%) — 明确的功能区分隔
悬浮/弹窗:     #ffffff + shadow — 最高层级
```

### 关键洞察

相邻区域的灰度差只需要 2-4% 就能被感知。不需要强烈的对比来区分区域。

## 彩色使用规则

### 60-25-10-5 法则

| 占比 | 角色 | 用途 |
|------|------|------|
| 60% | 灰阶 | 背景、边框、分隔 |
| 25% | 主色调 | 品牌色、导航、大面积色块 |
| 10% | 辅助色 | 次要按钮、标签、图标 |
| 5% | 强调色 | CTA、通知、关键状态 |

### 实际操作中的简化

多数界面只需要：
- **1 个主色**（品牌/主操作）
- **1 个功能色**（成功绿/警告黄/错误红 — 固定语义）
- **灰阶做其余一切**

不需要"调色盘生成器"给你 5-7 个颜色。颜色越少，每个颜色越有力量。

## 色彩与情绪

### 暖灰 vs 冷灰

同样的灰度，色温不同，氛围完全不同：

```
冷灰（蓝调）:  #f0f2f5 → 科技、专业、冷静
暖灰（黄调）:  #f5f3f0 → 温暖、人文、手工感
中性灰:        #f5f5f5 → 最安全的选择
```

选择灰调时，与主色互补：
- 主色偏暖（橙/红/黄）→ 用冷灰做背景，互相衬托
- 主色偏冷（蓝/紫/青）→ 用暖灰做背景，避免全冷调

### 明度 > 色相

需要"多个颜色"时，优先用同一色相的不同明度，而非不同色相：

```
# 与其这样（3 个色相）：
蓝 #2196F3 + 绿 #4CAF50 + 橙 #FF9800

# 不如这样（1 个色相的 3 个明度）：
深蓝 #1565C0 + 中蓝 #42A5F5 + 浅蓝 #BBDEFB
```

## 暗色模式的色彩逻辑

暗色模式不是"把白变黑"。是重新建立灰度梯度：

```
最底层背景:    #121212  — 不是纯黑 #000
卡片/面板:     #1e1e1e  — 比底层略亮
侧边栏:        #1a1a1a
工具栏:        #2d2d2d
文字:           #e0e0e0  — 不是纯白 #fff（太刺眼）
```

彩色在暗色模式下需要降低饱和度 10-20%，否则会过于刺眼。

## 常见错误

- ❌ 用纯黑 #000000 做文字色 — 太重，用 #222 或 rgba(0,0,0,0.87)
- ❌ 用纯白 #ffffff 做大面积背景 — 太亮，用 #fafafa
- ❌ 多于 3 个彩色 — 每多一个颜色，信息层级复杂度翻倍
- ❌ 状态色和品牌色撞色 — 绿色品牌 + 绿色"成功"会混淆语义
- ❌ 渐变当背景 — 除非有明确的审美意图（如 glassmorphism），否则纯色更安全
