# 空间密度与呼吸感

## 核心观点

留白不是"空"，是有功能的设计元素。留白引导视线、建立分组、创造节奏。

同时，过度留白和过度拥挤一样有害 — 前者浪费注意力，后者压垮注意力。

## 间距系统

### 8px 基础网格

所有间距基于 8px 的倍数。不是随意的 10px、15px、23px。

```
4px   — 极小间距：图标与标签、密切相关的行内元素
8px   — 紧凑间距：列表项内部、输入框内 padding
12px  — 适中间距：卡片内部 padding（紧凑型）
16px  — 标准间距：卡片内部 padding（标准型）
24px  — 宽松间距：区块间分隔、卡片间 gap
32px  — 大间距：主要区块间
48px  — 段落级间距：页面级区块分隔
64px  — 最大间距：首屏/封面级留白
```

### 内紧外松原则

元素内部紧凑，元素之间宽松。这创造了"岛屿"效果 — 每个功能区是一个紧密的岛，岛与岛之间有明确的海洋。

```
卡片内 padding:  12-16px （紧凑 — 内容紧密关联）
卡片间 gap:      24-32px （宽松 — 明确区分不同卡片）
区块间 margin:   48-64px （大间距 — 主题切换的呼吸感）
```

## 分隔策略

### 优先级（从优雅到粗暴）

1. **间距** — 最优雅。不需要任何视觉元素，仅靠距离区分
2. **背景色差异** — 相邻区域用微妙的灰度差（2-4%）自然分区
3. **分隔线** — 必须用时保持极细（1px）和极淡（opacity 0.1）
4. **边框** — 最后手段。卡片可以用，但能用阴影代替就用阴影

### 分隔线使用规则

如果非要用分隔线：
```css
.divider {
  border: none;
  height: 1px;
  background: rgba(0, 0, 0, 0.06);  /* 亮色主题 */
  /* background: rgba(255, 255, 255, 0.06); 暗色主题 */
  margin: 0 16px;  /* 不要全宽，留出两端呼吸 */
}
```

## 信息密度校准

### 密度类型

| 密度 | 特征 | 适用场景 |
|------|------|---------|
| 高密度 | 小字号、紧间距、多列 | 数据表格、仪表盘、代码编辑器 |
| 中密度 | 标准字号、标准间距 | 大多数应用界面 |
| 低密度 | 大字号、大间距、单列 | 落地页、文章、展示页 |

### 密度一致性

一个页面内，密度应该一致。不要在同一页面混用高密度表格和低密度卡片 — 视觉节奏会混乱。

例外：首屏（Hero）可以用低密度，下方内容用中密度 — 这是有意的节奏变化。

## 滚动条设计

滚动条是最容易被忽略的界面元素，但它显著影响整体质感。

```css
/* 自定义滚动条 */
::-webkit-scrollbar {
  width: 6px;
}
::-webkit-scrollbar-track {
  background: transparent;
}
::-webkit-scrollbar-thumb {
  background: rgba(0, 0, 0, 0.15);
  border-radius: 3px;
}
::-webkit-scrollbar-thumb:hover {
  background: rgba(0, 0, 0, 0.25);
}

/* Firefox */
* {
  scrollbar-width: thin;
  scrollbar-color: rgba(0, 0, 0, 0.15) transparent;
}
```

## 响应式间距

间距不是固定值，应随视口缩放：

```css
:root {
  --space-base: clamp(12px, 2vw, 16px);
  --space-lg: clamp(24px, 4vw, 32px);
  --space-xl: clamp(32px, 6vw, 64px);
}
```

## 常见错误

- ❌ 间距值随机（10px, 15px, 23px 混用）— 必须基于统一网格
- ❌ 全宽分隔线 — 两端留 margin，或直接用间距代替
- ❌ 所有间距都一样大 — 没有层级就没有分组
- ❌ 卡片内 padding 比卡片间 gap 还大 — 违反内紧外松原则
- ❌ 忘记滚动条样式 — 默认滚动条会破坏整体质感
- ❌ 移动端使用桌面端间距 — 间距应随视口缩放
